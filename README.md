# Kinematic Simulation Dataset of an Autonomous Pruning Tractor Robot for Takagi-Sugeno Fuzzy Modeling

This repository contains the dataset and metadata used in the scientific article: 
*"Fuzzy model identification and trajectory control for agricultural tractor robots: an optimal hybrid methodology."*

---

## Overview

This dataset includes the raw kinematic telemetry data used for the fuzzy identification and modeling of an autonomous pruning tractor. The target architecture is a hybrid fuzzy model based on Takagi-Sugeno (T-S) fuzzy inference systems, where cluster centers and antecedents are identified using the Fuzzy C-Means (FCM) clustering algorithm.

The raw data was collected from trajectory-tracking simulations in a MATLAB environment based on a non-linear kinematic bicycle model with Ackerman steering. The vehicle's steering was guided by a geometric Pure Pursuit algorithm, coupled with a PI controller for linear velocity and a PD controller for orientation.

The ultimate purpose of this dataset is to identify two sub-models:
1. A T-S fuzzy model to estimate the next-step X-displacement ($\Delta X(k+1)$).
2. A T-S fuzzy model to estimate the next-step Y-displacement ($\Delta Y(k+1)$).

Together, these models predict the global 2D coordinates $(X, Y)$ of the mobile robot, while the heading angle ($\theta$) is computed analytically.

---

## Dataset Structure

All data is located in the `data/` folder in both `.csv` and `.xlsx` formats:
* `Autonomous_tractor_kinematics_dataset`: Contains 2,743 sequential data rows structured into 7 columns.
* `Training_Trajectory_Waypoints`: Contains the coordinate waypoints of the reference path.

### Columns in Kinematics Dataset:
| Column | Variable | Description | Unit |
| :--- | :--- | :--- | :--- |
| 1 | `Time (t)` | Simulation time steps | Seconds ($s$) |
| 2 | `Velocity (v)` | Actual linear velocity generated by the PI controller | $m/s$ |
| 3 | `Angle theta` | Heading angle ($\theta$) computed via the bicycle model | Radians ($rad$) |
| 4 | `Position X` | Global X-coordinate at current time step $k$ | Meters ($m$) |
| 5 | `Position Y` | Global Y-coordinate at current time step $k$ | Meters ($m$) |
| 6 | `Delta X` | Incremental displacement: $\Delta X(k) = X(k) - X(k-1)$ | Meters ($m$) |
| 7 | `Delta Y` | Incremental displacement: $\Delta Y(k) = Y(k) - Y(k-1)$ | Meters ($m$) |

> **Note on Fuzzy Identification:**
> The inputs for the T-S fuzzy modeling include current and past states ($V(k)$, $V(k-1)$, $\theta(k)$, and $\theta(k-1)$), derived by applying delays to columns 2 and 3. The target outputs correspond to the next-step future displacements ($\Delta X(k+1)$ and $\Delta Y(k+1)$), obtained by shifting columns 6 and 7 forward by one time step ($k+1$).

---

## Steps to Reproduce

1. **Kinematic Simulation Setup:** Execute the MATLAB simulation environment based on the classical kinematic bicycle model.
2. **Trajectory Tracking Control:** Run the simulation coupling the Pure Pursuit algorithm (with PD control for orientation) and the PI controller (for linear velocity).
3. **Data Acquisition:** Sample the kinematic telemetry variables at a fixed time interval to obtain the 2,743 data points.
4. **Feature Extraction:** Extract global coordinates and numerically compute the differential translational variables ($\Delta X$ and $\Delta Y$).
